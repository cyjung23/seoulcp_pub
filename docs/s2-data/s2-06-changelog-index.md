## 운영 규칙

### 롤오버 규칙
- 활성 파일이 **30K 토큰 초과** 시 가장 오래된 4~5개 버전을 아카이브로 이동
- 활성 파일은 항상 **최근 3~5개 버전** 유지를 원칙으로 함
- 아카이브 이동 시 인덱스 파일 갱신 필수

### 5종 보고서 차등 갱신 규칙
신규 기획팀의 부담 경감을 위해 작업 유형별로 갱신 범위를 다르게 적용합니다.

| 작업 유형 | 갱신 대상 |
|-----------|-----------|
| 신규 기능/메이저 릴리즈 | 5종 모두 (README + changelog + priority + summary-table + milestones) |
| 버그 픽스 / 데이터 정리 | changelog + summary-table |
| 모니터링 보고 | changelog 만 |
| 보고서 한정 수정 | 해당 파일 만 |
